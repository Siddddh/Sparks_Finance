# Pre-Market Notifications — Setup & Deploy

A scheduled Cloud Function (`functions/index.js → premarketNotifications`) runs every trading day
at **08:00 America/New_York** and pushes each user a personalised digest of today's market-moving
events — earnings for the tickers in *their* portfolio + watchlist, plus market-wide macro events
(FOMC, CPI, jobs) and US market-holiday status. The web app shows the same digests live in the new
**Pre-Market** tab.

> **Data source note:** the original spec named Google Finance, which has **no public API**. The
> calendar data actually comes from the **Finnhub** free API (`/calendar/earnings`,
> `/calendar/economic`, `/stock/market-holiday`).

## What's in the repo

| Piece | File |

## One-time setup (required before it works)

1. **Upgrade billing.** Cloud Functions v2 + Cloud Scheduler need the **Blaze** plan on project
   `claude-apps-a6fe1`. (Free-tier quotas still apply; cost for one daily run is ~$0.)

2. **VAPID web-push key.** Firebase Console → Project Settings → **Cloud Messaging** → *Web Push
   certificates* → generate a key pair → copy the **public** key. Paste it into
   `firebase_hosting/index.html`, replacing `REPLACE_WITH_WEB_PUSH_PUBLIC_KEY` in:
   ```js
   const VAPID_PUBLIC_KEY = 'REPLACE_WITH_WEB_PUSH_PUBLIC_KEY';
   ```

3. **Finnhub API key.** Sign up free at https://finnhub.io, then store the key as a secret:
   ```sh
   cd KV/functions
   firebase functions:secrets:set FINNHUB_API_KEY    # paste the key when prompted
   ```

4. **Install function deps** (first deploy does this automatically, or locally):
   ```sh
   cd KV/functions && npm install
   ```

## Deploy

```sh
cd KV
firebase deploy --only firestore:rules,hosting,functions --project claude-apps-a6fe1
```

> `deploy.sh` runs a full `firebase deploy`, which now includes functions too — that's fine once the
> Finnhub secret and Blaze billing are set. **Do not** run `build_hosted_dashboard.py` — it overwrites
> the live `index.html`.

## Verify

1. **Token registration** — open the app, sign in, go to **Pre-Market** → click **Enable
   notifications**, accept the browser prompt. Confirm `users/{uid}.fcmTokens` now holds a token, and
   DevTools → Application → Service Workers shows `firebase-messaging-sw.js` active.
2. **Run the logic without waiting for 8 AM** — hit the test endpoint (replace KEY):
   ```
   https://us-central1-claude-apps-a6fe1.cloudfunctions.net/runPremarketNow?token=KEY
   ```
   It returns `{ ok, summary:{ users, sent, failed, skipped, prunedTokens } }`, writes
   `notifications/{uid}/list/*`, and (if the app is backgrounded) fires a system push.
3. **Live feed** — with the app open, the new digest appears in the Pre-Market tab via the
   `onSnapshot` listener and an in-app toast; the nav shows an unread badge. Click a card → it marks
   read (`read:true` in Firestore).
4. **Schedule** — `gcloud scheduler jobs list` shows a job at `0 8 * * 1-5` `America/New_York`.

## PWA / "Add to Home" (Phase 2)

The app is an installable Progressive Web App, which is what makes the 8 AM pushes reliable when the
browser is closed — and is **required for web push on iOS** (Safari only delivers push to installed
PWAs). Pieces:
- `firebase_hosting/manifest.webmanifest` — app name, icons, standalone display, theme `#1a73e8`.
- `firebase_hosting/icons/*` + `favicon.ico` — generated by `make_icons.py` (re-run
  `python make_icons.py` to regenerate; PNGs are committed so deploy needs no build step).
- `firebase-messaging-sw.js` — now also caches the app shell (`install`/`activate`/`fetch`,
  same-origin only) for offline + installability, alongside the FCM push handler.
- `index.html` — manifest/Apple meta tags, registers the SW on load, and shows an **Add to Home**
  button + install banner (`beforeinstallprompt`), with an **iOS** instructions sheet fallback.

Install flow: Android/desktop Chrome/Edge show the Install button/banner → tap → installed. iOS:
tap **Add to Home** → follow the Share → *Add to Home Screen* steps → open from the home screen and
enable notifications there.

Deploy Phase 2 (hosting only — no functions change):
```
firebase deploy --only hosting --project claude-apps-a6fe1
```

## Security note

`runPremarketNow` is a convenience test endpoint guarded only by the shared Finnhub token. **Remove
it or replace the guard with proper auth** before treating this as production.
